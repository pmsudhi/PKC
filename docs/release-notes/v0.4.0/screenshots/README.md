# v0.4.0 ‚Äî Screenshots

These PNGs are generated by Playwright from the Vite dev server with
canned MCP / Tauri fixtures ‚Äî no live daemon, no real account data.
Re-run them at any time:

```bash
cd ui
npx playwright test e2e/release-notes-screenshots.spec.ts --workers=1
```

The spec lives at `ui/e2e/release-notes-screenshots.spec.ts` and uses the
same `tauri-mock` fixture as the rest of the e2e suite.

## How the live shots are produced

1. Playwright boots `vite` (the UI dev server on port 1420).
2. Every test installs a `tauri-mock` fixture that stubs the Tauri IPC
   bridge ‚Äî the UI sees the mock as if a real daemon were responding.
3. The fixture data (Alice / Bob / Carol / Devon / Eva / Sam, plus
   tag / group / smart-group / rule rows) is hand-crafted in the spec
   to match the shapes the daemon would return on a real install.
4. After navigating to the route, the spec waits for the lead element
   to appear, then for a short settle delay (fcose layout, list
   virtualization), then takes a viewport screenshot at 1440√ó900.
5. PNGs land directly under `docs/release-notes/v0.4.0/screenshots/`.

The whole pass takes ~50 seconds with one worker. Adding new shots
is a few lines of spec ‚Äî copy an existing test, change the route +
filename, optionally extend the dispatcher to seed new MCP shapes.
